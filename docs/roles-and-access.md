# Роли и права доступа

## Роли

- `viewer`
  - просмотр складов, товаров, категорий и истории;
  - без права создавать, редактировать и выполнять операции.
- `cashier`
  - все права `viewer`;
  - может выполнять только расход.
- `warehouse_manager`
  - управляет складами, товарами, категориями;
  - выполняет приход, расход и перемещение.
- `super_admin`
  - все права `warehouse_manager`;
  - управляет пользователями.

## Матрица прав

| Право | viewer | cashier | warehouse_manager | super_admin |
|---|---|---|---|---|
| Просмотр складов | Да | Да | Да | Да |
| Создание/редактирование складов | Нет | Нет | Да | Да |
| Архивация складов | Нет | Нет | Да | Да |
| Просмотр товаров | Да | Да | Да | Да |
| Создание/редактирование товаров | Нет | Нет | Да | Да |
| Архивация товаров | Нет | Нет | Да | Да |
| Просмотр категорий | Да | Да | Да | Да |
| Создание/редактирование категорий | Нет | Нет | Да | Да |
| Архивация категорий | Нет | Нет | Да | Да |
| Приход | Нет | Нет | Да | Да |
| Расход | Нет | Да | Да | Да |
| Перемещение | Нет | Нет | Да | Да |
| Просмотр истории | Да | Да | Да | Да |
| Управление пользователями | Нет | Нет | Нет | Да |

## Где обеспечивается доступ

- UI-уровень:
  - `RoleGate`;
  - `usePermission`;
  - защита маршрута `/users`.
- База данных:
  - RLS policies на таблицах;
  - RPC-функции дополнительно валидируют роль перед изменением остатков.
- Серверные функции:
  - `admin-create-user`;
  - `admin-update-user-password`;
  - обе доступны только активному `super_admin`.

## Важные ограничения

- деактивированный пользователь автоматически разлогинивается и не может продолжать работу;
- последнего активного `super_admin` нельзя деактивировать через UI;
- складские операции нельзя выполнять прямым `insert/update` в таблицы остатков и операций, только через RPC;
- загрузка фото товара доступна только `warehouse_manager` и `super_admin`.
