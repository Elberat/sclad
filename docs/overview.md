# Обзор системы

## Назначение

`Sclad` это single-tenant система учета склада и движения товаров. Приложение позволяет:

- вести справочник складов;
- вести номенклатуру и категории товаров;
- хранить остатки по каждому товару в разрезе складов;
- выполнять приход, расход и перемещение;
- видеть журнал операций;
- управлять пользователями и ролями.

## Бизнес-область

Система покрывает базовый контур оперативного складского учета:

- учет мест хранения;
- учет карточек товаров;
- контроль остатков;
- регистрацию движений;
- разграничение доступа по ролям.

Система не реализует:

- многокомпанейность;
- заказы клиентов и поставщиков;
- резервирование;
- инвентаризацию и корректировки;
- закупочные документы и финансовый документооборот;
- партии, серийные номера, сроки годности;
- печатные формы.

## Основные бизнес-сущности

- `Склад`:
  - место хранения товара;
  - может быть активным или архивным.
- `Категория`:
  - справочник для группировки товаров.
- `Товар`:
  - карточка номенклатуры;
  - хранит атрибуты, цены и фото;
  - не хранит остаток внутри себя.
- `Остаток`:
  - количество конкретного товара на конкретном складе.
- `Операция`:
  - факт изменения остатков;
  - типы: `receipt`, `sale`, `transfer`.
- `Пользователь`:
  - учетная запись с ролью и признаком активности.

## Архитектура на верхнем уровне

- Frontend:
  - React + TypeScript + Vite;
  - React Router;
  - React Query;
  - формы на `react-hook-form` + `zod`;
  - UI на shadcn/radix.
- Backend:
  - Supabase Auth;
  - Postgres;
  - Row Level Security;
  - RPC-функции для операций с остатками;
  - Edge Functions для администрирования пользователей;
  - Storage bucket для фотографий товаров.
