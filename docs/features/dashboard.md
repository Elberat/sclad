# Фича: Дашборд

## Бизнес-цель

Дать пользователю быстрый вход в систему:

- увидеть текущее состояние справочников и остатков;
- запустить основную складскую операцию;
- увидеть последние движения.

## Пользовательские сценарии

- пользователь открывает приложение после логина и попадает на дашборд;
- видит количество активных складов, товаров, категорий и суммарный остаток;
- запускает приход, расход или перемещение из быстрых действий;
- переходит в полную историю операций.

## Системная реализация

Экран агрегирует данные из:

- `warehouses` с фильтром `is_active = true`;
- `items` с фильтром `is_active = true`;
- `item_categories` с фильтром `is_active = true`;
- `inventory_balances` для суммарного количества;
- `inventory_operations` для последних 10 операций.

Быстрые действия показываются по ролям:

- приход: `canDoReceipt`;
- расход: `canDoSale`;
- перемещение: `canDoTransfer`.

## Бизнес-правила

- на дашборде считаются только активные справочники;
- суммарный остаток считается как сумма всех строк `inventory_balances`;
- последние операции показываются в порядке `created_at desc`.

## Ограничения

- дашборд не содержит аналитики по деньгам, только по количеству;
- нет drill-down по метрикам кроме переходов на списки и историю;
- нет пользовательской настройки виджетов.
