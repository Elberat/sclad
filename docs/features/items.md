# Фича: Товары

## Бизнес-цель

Вести карточки номенклатуры и контролировать остатки по складам.

## Пользовательские сценарии

- создать товар;
- указать категорию, модель, SKU, описание и цены;
- загрузить фото;
- открыть карточку товара;
- увидеть остатки по складам;
- выполнить приход, расход или перемещение по конкретному товару;
- архивировать и восстановить товар.

## Системная реализация

Список товаров:

- читает `items`;
- подтягивает `item_categories(name)`;
- подтягивает `inventory_balances(quantity, warehouse_id)`;
- считает `total_quantity` на frontend;
- фильтрует по:
  - поиску;
  - категории;
  - складу;
  - наличию;
  - статусу.

Карточка товара:

- читает данные товара;
- читает остатки по складам;
- читает последние операции по товару.

Форма товара:

- валидирует обязательные поля `name` и `category_id`;
- использует активные категории;
- позволяет загрузить фото в bucket `item-photos`;
- сохраняет `image_url` в карточку товара.

## Бизнес-правила

- товар можно создавать и изменять только ролями `warehouse_manager` и `super_admin`;
- в операциях можно использовать только активные товары;
- фото может загружать только роль с правами менеджера склада или выше;
- архивный товар остается в истории и остатках, но исключается из активных сценариев выбора.

## Важные детали аналитики

- остаток не хранится в карточке товара, а вычисляется по таблице `inventory_balances`;
- цены не участвуют в серверных расчетах и пока играют только справочную роль;
- `specs_json` присутствует в схеме, но в UI сейчас не используется;
- список товаров показывает суммарный остаток по всем складам.

## Ограничения и риски

- нет валидации формата SKU и цен;
- нет уникальности по SKU;
- нет удаления старых фотографий из Storage после замены URL в карточке;
- архивирование товара не проверяет наличие остатков и допускается при ненулевом балансе.
