# Фича: Категории

## Бизнес-цель

Классифицировать товары для фильтрации и визуального порядка в справочнике номенклатуры.

## Пользовательские сценарии

- создать категорию;
- изменить название и описание;
- архивировать категорию;
- восстановить категорию;
- выбирать активную категорию при создании или редактировании товара.

## Системная реализация

- список категорий строится по таблице `item_categories`;
- есть фильтр по статусу `active / archived`;
- форма используется и для создания, и для редактирования;
- для формы товара подгружаются только активные категории.

## Бизнес-правила

- управлять категориями могут только `warehouse_manager` и `super_admin`;
- архивная категория скрывается из выбора в форме товара;
- архивирование не удаляет связь у уже существующих товаров.

## Ограничения и риски

- нет проверки, что у архивируемой категории есть связанные товары;
- нет уникальности названия категории на уровне схемы;
- нет иерархии категорий, только плоский справочник.
