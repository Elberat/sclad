# Фича: Склады

## Бизнес-цель

Поддерживать справочник мест хранения и видеть остатки по каждому складу.

## Пользовательские сценарии

- создать склад;
- изменить название и описание склада;
- открыть карточку склада;
- посмотреть товары и остатки на складе;
- посмотреть последние операции по складу;
- выполнить приход, расход или перемещение из карточки склада;
- архивировать пустой склад;
- восстановить архивный склад.

## Системная реализация

Список складов:

- загружает `warehouses`;
- считает количество позиций через связь `inventory_balances(count)`;
- поддерживает фильтр `active / archived / all`.

Карточка склада:

- загружает сам склад;
- загружает остатки `inventory_balances` по `warehouse_id`;
- загружает последние операции, где склад участвует как источник или получатель;
- поддерживает локальный поиск по названию, модели и SKU товара.

## Бизнес-правила

- склад можно создать и редактировать только ролями `warehouse_manager` и `super_admin`;
- архивировать склад можно только если на нем нет положительных остатков;
- архивный склад исключается из активных списков выбора для операций;
- операции на карточке склада ограничиваются правами текущего пользователя.

## Важные детали аналитики

- количество позиций на складе это количество строк остатков, а не сумма единиц товара;
- архивность склада управляется через `is_active` и `archived_at`;
- история склада формируется по двум полям:
  - `source_warehouse_id`;
  - `destination_warehouse_id`.

## Ограничения и риски

- нет жесткой серверной проверки "нельзя архивировать склад с остатками" в SQL-миграциях, сейчас блокировка реализована на UI;
- в таблице остатков на карточке склада строка не ведет на карточку товара на desktop, переход есть только в mobile-версии списка и в других разделах;
- удаление склада физически не используется, только soft archive.
