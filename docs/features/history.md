# Фича: История операций

## Бизнес-цель

Дать единый журнал всех движений товара для контроля, аудита и поиска событий.

## Пользовательские сценарии

- открыть полный журнал;
- отфильтровать по складу;
- отфильтровать по товару;
- отфильтровать по типу операции;
- отфильтровать по периоду;
- передать ссылку на уже отфильтрованный журнал другому сотруднику;
- перейти в карточку товара из строки операции.

## Системная реализация

- данные берутся из `inventory_operations`;
- подгружаются связи:
  - товар;
  - склад-источник;
  - склад-получатель;
  - пользователь;
- фильтры хранятся в query params URL;
- реализована пагинация по 50 записей;
- сортировка по `created_at desc`.

## Бизнес-правила

- если выбран склад, показываются операции, где он участвует как источник или получатель;
- если выбран товар, журнал сужается до конкретной номенклатуры;
- фильтр периода работает по `created_at`.

## Аналитические замечания

- журнал хранит факт движения, но не хранит снимок остатка до и после операции;
- в расходе заполнен только склад-источник;
- в приходе заполнен только склад-получатель;
- в перемещении заполнены оба склада.

## Ограничения и риски

- нет экспорта в Excel/CSV;
- нет фильтра по пользователю;
- нет поиска по комментарию;
- нет детального документа операции, только запись в журнале.
