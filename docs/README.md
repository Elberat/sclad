## Как читать

- `overview.md` нужен для бизнес-описания продукта и границ системы.
- `roles-and-access.md` фиксирует матрицу ролей и ограничения.
- `data-model.md` описывает сущности, связи, RLS, RPC и Edge Functions.
- `features/*` разбивает систему на прикладные фичи. Внутри каждой фичи есть:
  - бизнес-цель;
  - пользовательские сценарии;
  - системная реализация;
  - ограничения и риски;
  - точки развития.
